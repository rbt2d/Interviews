## Interview Process — Full Details

### Stage 1: Technical Assessment (SCHEDULED ✅)
- **Format:** 1-hour Zoom
- **Days available:** Monday, Wednesday, or Friday
- **What to expect:** New system design question, **train-related scenario** (e.g., sensors, real-time tracking, scheduling)
- **What they want:** Ask clarifying questions, think out loud, diagram the system live
- **Pass rate:** ~95% — Luanne described it as almost a formality if you engage well
- **Key tip:** The best candidates ask lots of clarifying questions and verbalize their reasoning

### Stage 2: Full Loop (Post-Tech Assessment)
Scheduled on a single day (or split Mon/Tue or Wed/Fri if needed). Each round is **45 minutes with 15 minutes between**.

| Round | Type | Details |
|-------|------|---------|
| Coding 1 | Data Structures & Algorithms | Logical and maintainable code |
| Coding 2 | Data Structures & Algorithms | Same format, different problem |
| New System Design | Open-ended system design | Train-related, similar to tech assessment but deeper |
| Existing System Design | Walk through a system you built | Your choices, your architecture, your tradeoffs |
| Behavioral | Hiring manager / team leader | Project ownership, ambiguity, staff-level bar |

**Interviewers:** Principal/Staff level engineers + a hiring manager or team leader for behavioral

## Prep Priorities Before Technical Assessment

### 1. System Design (Most Critical for Tech Assessment)
- **Train-related scenarios** — practice thinking through:
  - Real-time train location tracking system (GPS sensors → event stream → dashboard)
  - Railroad crossing safety system (sensor network → alert system)
  - Freight routing optimization (schedule + capacity + routing)
  - Train health monitoring (sensor anomaly detection pipeline)
- **Framework:** Ask clarifying questions first → estimate scale → define components → draw data flow → discuss tradeoffs
- The AT&T auth pipeline is perfect for the "existing system design" round

### 2. Coding — DSA (Biggest Risk Area)
- LeetCode medium difficulty: arrays, strings, graphs, trees, dynamic programming
- Preferred language: **Java** (also Python/Kotlin/Go)
- Focus on: clean, readable, maintainable code — not just getting the answer
- Practice explaining your approach out loud while coding

### 3. Staff vs Senior Level Differentiator
- Luanne's key intel: staff level = **handling ambiguity**
- When given an undefined problem, do you ask the right clarifying questions?
- Show structured thinking: *"Before I start designing, I have a few clarifying questions…"*
- In behavioral: show examples of owning multi-quarter projects with ambiguous requirements
